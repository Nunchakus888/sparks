# Sparks - AI 学习资源导航

## 📚 项目简介

本项目收集整理了 AI 工具学习资源，专注于帮助**非技术人员**从零开始掌握 AI 工具的使用。

---

## 🏁 最终目标

学完本项目，你将能够：

- **独立使用 AI 工具**处理日常工作（文档整理、数据分析、内容生成）
- **用自然语言描述任务**，让 AI 自动完成（不需要写代码）
- **设计自己的工作流**，把重复性任务交给 AI（如英语学习、文件整理、报告生成）
- **节省 50%+ 的时间**在文档处理和内容创作上

## 📖 术语表（非技术人员必读）

> 文档中会出现一些专业术语，这里用大白话解释清楚。看不懂的时候回来查这个表。

### 基础概念

#### AI / 人工智能
**英文全称**：Artificial Intelligence

**是什么**：能像人一样思考和工作的计算机程序。

**举例**：
- ChatGPT 能回答问题、写文章
- 本项目的 AI 能读 PDF、生成学习材料

**类比**：就像一个永远不累的助手，你告诉它要做什么，它就帮你做。

---

#### API / 接口
**英文全称**：Application Programming Interface（应用程序编程接口）

**是什么**：连接你的工具和 AI 大脑的"电话线"。

**举例**：
- 你在 Open-Cowork 中输入任务
- Open-Cowork 通过 API 把任务发给智谱 GLM
- 智谱 GLM 处理完，通过 API 把结果返回

**类比**：
- API = 餐厅的服务员
- 你（顾客）告诉服务员要什么菜
- 服务员去厨房（AI）下单
- 厨房做好后，服务员端给你

**为什么需要**：AI 模型（如智谱 GLM）在云端服务器上，API 是你和它沟通的桥梁。

**相关链接**：
- [什么是 API？（知乎）](https://www.zhihu.com/question/38594466)
- [API 入门教程（阮一峰）](https://www.ruanyifeng.com/blog/2019/04/api-design.html)

---

#### API Key / 密钥
**是什么**：你的"身份证号"，证明你有权使用 AI 服务。

**举例**：
- 注册智谱 GLM 后，会给你一串字符：`abc123.xyz789`
- 这就是你的 API Key
- 配置到 Open-Cowork 后，它就能代表你调用 AI

**类比**：
- API Key = 银行卡密码
- 有了它，工具才能代表你使用 AI 服务
- 要保密，不要泄露给别人

**如何获取**：
1. 注册 AI 模型账号（如智谱 GLM）
2. 进入控制台 → API Keys
3. 点击"创建新密钥"
4. 复制保存（只显示一次）

**相关链接**：
- [智谱 GLM API Key 获取教程](https://open.bigmodel.cn/dev/howuse/introduction)

---

#### Token / 令牌
**是什么**：AI 处理文本的计量单位，类似"字数"。

**举例**：
- 英文：1 个单词 ≈ 1-2 个 token
- 中文：1 个汉字 ≈ 1.5-2 个 token
- "Hello world" ≈ 2 tokens
- "你好世界" ≈ 6 tokens

**为什么重要**：AI 服务按 token 收费。

**举例**：
- 智谱 GLM：¥0.1/1000 tokens
- 处理一本 300 词的绘本 ≈ 500 tokens ≈ ¥0.05

**类比**：
- Token = 出租车的公里数
- 处理的文本越多，token 越多，费用越高

**相关链接**：
- [什么是 Token？（OpenAI 官方）](https://platform.openai.com/tokenizer)

### 高级概念（可选）

#### MCP / Model Context Protocol
**英文全称**：Model Context Protocol（模型上下文协议）

**是什么**：让 AI 连接外部工具的"插件系统"。

**举例**：
- 浏览器 MCP：让 AI 能打开网页、点击按钮
- 数据库 MCP：让 AI 能查询数据库
- Notion MCP：让 AI 能读写 Notion 笔记

**类比**：
- AI = 手机
- MCP = 手机 App
- 装了不同的 App，手机能做不同的事

**是否必须**：❌ 初学者不需要，基础功能够用了。

**相关链接**：
- [MCP 官方文档](https://modelcontextprotocol.io/)
- [MCP 入门教程](https://modelcontextprotocol.io/introduction)

#### Prompt / 提示词
**是什么**：你给 AI 的指令或问题。

**举例**：
- 简单 Prompt："帮我写一篇作文"
- 详细 Prompt："帮我写一篇 500 字的作文，主题是春天，要包含 3 个比喻句"

**为什么重要**：Prompt 写得越清楚，AI 的结果越好。

**技巧**：
- ✅ 说清楚要做什么
- ✅ 说清楚输出格式
- ✅ 给出具体例子
- ❌ 不要太模糊

**举例对比**：

| 不好的 Prompt | 好的 Prompt |
|--------------|------------|
| "处理这个文件" | "从 books/xxx.pdf 提取文本，生成词汇表，保存为 vocabulary.md" |
| "帮我学英语" | "从这本绘本提取 20 个核心单词，包含音标、释义、例句" |

**相关链接**：
- [Prompt 工程指南](https://www.promptingguide.ai/zh)
- [如何写好 Prompt？（知乎）](https://www.zhihu.com/question/584402332)

#### PDF / Portable Document Format
**英文全称**：Portable Document Format（便携式文档格式）

**是什么**：一种文件格式，能保持排版不变。

**特点**：
- ✅ 跨平台（Windows、Mac、手机都能打开）
- ✅ 排版固定（不会乱）
- ❌ 不好编辑

**本项目用途**：从 PDF 绘本提取文本，生成学习材料。

**注意**：
- ✅ 文字版 PDF：可以直接提取文字
- ❌ 扫描版 PDF：需要 OCR（文字识别）

---

#### Markdown / MD
**英文全称**：Markdown（标记语言，无完整全称）

**是什么**：一种简单的文本格式，用符号表示排版。

**举例**：
```markdown
# 标题
## 二级标题
**粗体**
- 列表项 1
- 列表项 2
```

**为什么用它**：
- ✅ 简单易学
- ✅ 纯文本，任何编辑器都能打开
- ✅ 可以转换为 HTML、PDF

**本项目用途**：生成的词汇表、练习题都是 Markdown 格式。

**查看工具**：
- macOS：Typora、Obsidian
- Windows：Typora、VS Code
- 在线：https://markdown-it.github.io/

**相关链接**：
- [Markdown 入门教程](https://www.runoob.com/markdown/md-tutorial.html)
- [Markdown 在线编辑器](https://markdown-it.github.io/)

---

#### TTS / Text-to-Speech / 文字转语音
**英文全称**：Text-to-Speech（文本转语音）

**是什么**：把文字转换成语音的技术。

**举例**：
- 输入："Hello world"
- 输出：一个 MP3 文件，读出"Hello world"

**本项目用途**：为单词和句子生成英语朗读音频。

**方案对比**：

**相关链接**：

---

#### OCR / Optical Character Recognition / 文字识别
**英文全称**：Optical Character Recognition（光学字符识别）

**是什么**：从图片中识别文字的技术。

**举例**：
- 输入：一张照片（上面有文字）
- 输出：提取出的文字

**什么时候需要**：
- 扫描版 PDF（图片格式）
- 手写笔记照片
- 截图中的文字

**工具推荐**：
- macOS：系统自带（右键 → 拷贝文本）
- Windows：微软 PowerToys（Text Extractor）
- 在线：https://www.onlineocr.net/

**真相**：没有傻问题，只有不敢问的人。所有高手都是从"傻问题"开始的

### 给自己的承诺

在开始学习前，请对自己说：

✅ "我允许自己不懂"
✅ "我允许自己犯错"
✅ "我允许自己问问题"
✅ "我允许自己慢慢来"

**记住**：学习 AI 工具不是考试，没有人给你打分。你的目标是让 AI 帮你解决实际问题，而不是成为技术专家。

## 🎓 学习小贴士

### 1. 设定小目标
不要想着"我要学会 AI"，而是：
- ✅ "今天我要完成第一个任务"
- ✅ "这周我要处理 3 本绘本"
- ✅ "这个月我要设计自己的工作流"

### 2. 记录进步
创建一个学习日志：
```markdown
## 学习日志

### 2026-03-03
- ✅ 完成阶段 1，了解了 AI 工具能做什么
- ✅ 注册了智谱 GLM 账号
- 📝 问题：API Key 是什么？（已查术语表，理解了）

### 2026-03-04
- ✅ 安装了 Open-Cowork
- ✅ 配置成功，跑通了第一个任务
- 🎉 成就：让 AI 帮我创建了一个待办清单！
```

### 3. 加入学习小组
- 找 1-2 个同样在学习的朋友
- 互相鼓励，分享进度
- 遇到问题一起讨论

### 4. 庆祝小胜利
- 完成第一个任务 → 给自己点个赞
- 处理第一本绘本 → 发个朋友圈
- 设计第一个工作流 → 请自己吃顿好的

### 5. 不要追求完美
- 第一次生成的结果不完美？正常！
- 任务描述写得不够好？慢慢改进！
- 有些功能还不会用？先用会基础的！

**记住**：完成比完美更重要。
