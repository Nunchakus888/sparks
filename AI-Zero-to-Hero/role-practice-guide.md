# 角色实战案例 - 运营 & 产品经理

## 🎯 这是什么？

针对**运营人员**和**产品经理**两个岗位，提供可直接复制使用的 AI 任务模板。
每个案例都是真实工作场景，不需要写代码，只需把任务描述粘贴到 Open-Cowork 中执行。

> 前置条件：已完成阶段 1-3，熟悉 Open-Cowork 基本操作。

## 📣 运营人员实战案例

### 案例 1：用户反馈分析报告

**场景**：你收集了一批用户反馈（App Store 评论、问卷、客服记录），需要快速提炼关键问题。

**准备**：将反馈内容整理为文本文件，放入工作目录。
- 支持格式：`.txt`、`.csv`、`.xlsx`
- 每条反馈一行，或按 CSV 格式整理

**任务模板**：
```
请分析 data/user-feedback.csv 中的用户反馈数据：

【分析维度】
1. 情感分析：正面 / 中性 / 负面，统计占比
2. 问题分类：按主题归类（如：性能、功能、体验、价格）
3. 高频关键词：提取出现频率最高的 20 个关键词
4. 核心问题 TOP 10：按提及频次排序，附原始反馈引用
5. 趋势判断：与上期对比（如有历史数据）

【输出要求】
- 保存为 reports/user-feedback-report.md
- 包含数据图表描述（柱状图、饼图的文字描述）
- 每个问题附 2-3 条原始用户反馈作为证据
- 最后给出 3-5 条改进建议

【格式】
- Markdown 格式
- 包含表格和列表
- 适合直接粘贴到飞书/钉钉文档
```

**预期输出**：
```markdown
# 用户反馈分析报告（2026年3月）

## 总体概况
- 总反馈数：523 条
- 正面：62%  中性：18%  负面：20%

## 问题分类
| 类别 | 数量 | 占比 | 典型反馈 |
|------|------|------|---------|
| 性能问题 | 45 | 8.6% | "加载太慢了，等了10秒" |
| 功能缺失 | 38 | 7.3% | "希望能支持导出PDF" |
| ...  | ... | ... | ... |

## 核心问题 TOP 10
1. 启动速度慢（提及 45 次）
   > "每次打开都要等好久"
   > "加载速度能不能优化一下"
2. ...

## 改进建议
1. 优先优化启动速度（影响面最大）
2. ...
```

### 案例 2：社交媒体内容批量生成

**场景**：需要为一个产品/活动生成多平台的推广内容。

**任务模板**：
```
请根据以下产品信息，生成社交媒体推广内容：

【产品信息】
- 产品名称：[你的产品名]
- 核心卖点：[1-3个核心卖点]
- 目标用户：[目标人群描述]
- 活动信息：[如有促销活动]

【生成内容】

1. 小红书笔记（3 篇）
   - 标题：带 emoji，吸引点击
   - 正文：300-500 字，口语化，分段清晰
   - 标签：10-15 个相关标签
   - 风格：种草型 / 测评型 / 教程型 各一篇

2. 公众号推文（1 篇）
   - 标题：3 个备选标题
   - 正文：800-1200 字
   - 结构：痛点引入 → 解决方案 → 产品介绍 → 行动号召

3. 朋友圈文案（5 条）
   - 每条 50-100 字
   - 风格各异：日常分享 / 痛点共鸣 / 数据说服 / 场景描述 / 限时紧迫

4. 抖音/视频脚本（1 个）
   - 时长：30-60 秒
   - 结构：开头钩子（3秒）→ 痛点 → 解决方案 → 行动号召
   - 包含画面描述和口播文案

【输出】
- 保存为 content/social-media-[日期].md
- 每个平台的内容用分割线隔开
```

### 案例 3：活动复盘报告

**场景**：一场线上/线下活动结束，需要快速生成复盘报告。

**任务模板**：
```
请根据 data/event-data.csv 生成活动复盘报告：

【活动基本信息】
- 活动名称：[名称]
- 活动时间：[日期]
- 活动类型：[线上直播/线下沙龙/促销活动]
- 目标：[如：拉新 500 人 / GMV 10万]

【分析内容】
1. 核心数据概览
   - 参与人数 vs 目标
   - 转化率（报名→参与→付费）
   - ROI（投入产出比）

2. 各环节漏斗分析
   - 曝光 → 点击 → 报名 → 参与 → 转化
   - 每个环节的转化率和流失原因

3. 亮点与不足
   - 做得好的 3 件事（附数据支撑）
   - 需要改进的 3 件事（附具体建议）

4. 用户反馈摘要
   - 正面反馈 TOP 5
   - 负面反馈 TOP 5

5. 下次活动建议
   - 可复用的经验
   - 需要调整的策略

【输出】
- 保存为 reports/event-review-[活动名].md
- 适合在周会上直接展示
```

### 案例 5：数据日报/周报自动生成

**场景**：每天/每周需要从数据中生成固定格式的报告。

**任务模板**：
```
请根据 data/daily-metrics.csv 生成运营日报：

【数据指标】
1. 核心指标
   - DAU / MAU
   - 新增用户数
   - 留存率（次日/7日/30日）
   - 付费转化率

2. 增长指标
   - 各渠道新增用户数
   - 渠道转化率对比
   - 获客成本（CAC）

3. 营收指标
   - 当日/当周 GMV
   - ARPU（每用户平均收入）
   - 付费用户数

【输出格式】
- 保存为 reports/daily-report-[日期].md
- 每个指标标注环比变化（↑↓）
- 异常数据用醒目标记
- 最后附"今日关注"（需要重点关注的 1-3 个数据变化）

【示例格式】
| 指标 | 今日 | 昨日 | 环比 | 备注 |
|------|------|------|------|------|
| DAU | 12,500 | 12,100 | ↑3.3% | 正常 |
| 新增用户 | 850 | 620 | ↑37% | ⚠️ 异常增长，需排查 |
```

## 📋 产品经理实战案例

### 案例 1：用户访谈记录整理

**场景**：做了 10 场用户访谈，需要从录音转录文本中提取需求。

**任务模板**：
```
请分析 data/interviews/ 文件夹中的用户访谈记录：

【任务】
1. 逐份阅读访谈记录
2. 提取每位用户提到的：
   - 痛点（当前遇到的问题）
   - 需求（希望有的功能/改进）
   - 使用场景（在什么情况下使用产品）
   - 情感倾向（满意/不满/中性）

3. 汇总分析：
   - 高频痛点 TOP 10（按提及人数排序）
   - 高频需求 TOP 10（按提及人数排序）
   - 用户画像归纳（2-3 类典型用户）
   - 关键引语摘录（每个痛点/需求附 1-2 条原话）

【输出】
- 保存为 reports/interview-insights.md
- 包含汇总表格和详细分析

【输出格式示例】
## 高频痛点 TOP 10

| 排名 | 痛点 | 提及人数 | 占比 | 典型用户原话 |
|------|------|---------|------|------------|
| 1 | 导出功能不好用 | 7/10 | 70% | "每次导出都要重新设置格式" |
| 2 | 搜索结果不准确 | 5/10 | 50% | "搜了半天找不到想要的" |
```

**场景**：需要为一个新功能撰写 PRD。

**任务模板**：
```
请帮我撰写以下功能的 PRD：

【功能概述】
- 功能名称：[名称]
- 一句话描述：[用一句话说清楚这个功能做什么]
- 目标��户：[谁会用这个功能]
- 解决的问题：[用户当前的痛点是什么]

【请生成以下内容】

1. 背景与目标
   - 业务背景（为什么要做）
   - 用户痛点（解决什么问题）
   - 产品目标（上线后期望达到什么效果）
   - 成功指标（用什么数据衡量成功）

2. 需求详述
   - 功能范围（做什么 / 不做什么）
   - 用户故事（As a... I want... So that...）

3. 交互说明
   - 页面结构（用文字描述页面布局）
   - 操作流程（步骤 1 → 步骤 2 → ...）

4. 数据需求
   - 需要记录的数据埋点
   - 需要展示的数据指标

5. 优先级与排期建议
   - P0（必须有）/ P1（应该有）/ P2（可以有）
   - 建议分几期迭代

【输出】
- 保存为 docs/prd-[功能名].md
- 格式清晰，适合发给开发团队
```

### 案例 4：需求优先级排序

**场景**：积压了一堆需求，需要理性排序，制定迭代计划。

**任务模板**：
```
请根据 data/backlog.csv 进行需求优先级排序：

【评估维度】（每项 1-5 分）
1. 用户价值：解决多少用户的问题？影响面多大？
2. 业务价值：对营收/增长/留存的贡献？
3. 开发成本：预估工作量（1=小 5=大）
4. 紧急程度：是否有时间窗口或外部压力？

【排序方法】
- 计算 ROI 分数 = (用户价值 + 业务价值) / 开发成本
- 紧急程度作为加权因子

【输出内容】

1. 需求优先级排序表
   | 排名 | 需求 | 用户价值 | 业务价值 | 开发成本 | 紧急度 | ROI | 建议 |
   |------|------|---------|---------|---------|--------|-----|------|

2. 迭代计划建议
   - 第 1 期（本月）：TOP 3 需求
   - 第 2 期（下月）：第 4-6 需求
   - 待定池：其余需求

3. 决策说明
   - 为什么这样排序
   - 需要注意的依赖关系
   - 风险提示

【输出】
- 保存为 docs/sprint-plan-[日期].md
```

## 💡 使用技巧

### 1. 任务描述的通用结构

不管什么岗位，好的任务描述都遵循这个结构：

```
【背景】我是什么角色，在做什么事
【输入】数据/文件在哪里，是什么格式
【任务】具体要做什么（分步骤列清楚）
【输出】保存到哪里，什么格式，给谁看
```

### 2. 迭代优化

第一次生成的结果不满意？追加描述即可：

```
上次生成的报告：
- 分析维度不够，请增加"用户生命周期"维度
- 建议部分太笼统，请给出具体的执行步骤
- 表格太长，请只保留 TOP 5
```

### 3. 建立模板库

把好用的任务描述保存下来：

```
~/AI-Workspace/
└── templates/
    ├── 运营/
    │   ├── 用户反馈分析.txt
    │   ├── 社交媒体内容.txt
    │   ├── 活动复盘.txt
    │   └── 竞品周报.txt
    └── 产品/
        ├── 访谈整理.txt
        ├── PRD模板.txt
        ├── 竞品分析.txt
        └── 需求排序.txt
```

每次使用时，复制模板 → 修改具体信息 → 粘贴执行。

### 4. 数据准备技巧

AI 处理数据的效果取决于数据质量：

| 数据格式 | 推荐程度 | 说明 |
|---------|---------|------|
| CSV | ⭐⭐⭐⭐⭐ | 最推荐，结构清晰 |
| TXT（每行一条） | ⭐⭐⭐⭐ | 简单场景够用 |
| Markdown 表格 | ⭐⭐⭐⭐ | 可读性好 |
| Excel (.xlsx) | ⭐⭐⭐ | 需要工具支持 |
| 纯文本（无结构） | ⭐⭐ | AI 能处理，但效果不稳定 |

**建议**：尽量用 CSV 格式，第一行写清楚列名。

## ⚠️ 注意事项

### 1. AI 生成内容需要审核
- AI 可能会"编造"数据，务必核实关键数字
- 分析结论需要结合你的业务经验判断
- 建议作为初稿，人工修改后再使用

### 2. 数据安全
- 不要将敏感数据（用户隐私、财务数据）直接发给 AI
- 使用前脱敏处理（去掉姓名、手机号等）
- 内部机密信息谨慎使用

### 3. 持续优化
- 每次使用后，记录哪些描述效果好
- 逐步完善你的模板库
- 不同场景可能需要不同的描述方式
